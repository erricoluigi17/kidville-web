# P3.3h — Proxy upload cartaceo reale (modulistica)

**Fase:** P3 · **Slice:** P3.3h · **DL:** DL-032 · **Migrazione:** 20260749
**Branch:** feat/master-plan-registro

## Obiettivo
Rendere **reale** il proxy upload cartaceo: la maestra/Segreteria carica la **scansione** del modulo
firmato a penna consegnato a scuola, lo **storage effettivo** del file, con **gate** ed **evidenza**
strutturata. (Sistema legacy `forms_templates`/`forms_submissions`, autorizzazioni per classe.)

## Contesto (verificato)
- `POST /api/teacher/modulistica` (proxy) è uno **stub**: accetta `file_path` come **stringa** (nessun
  upload reale), è **ungated** (`teacher_id` dal body + DEFAULT), `signature_log` ad-hoc, answers con typo.
- Merge PDF classe **esiste** (`/api/admin/documents-merge` + `handleExportMergePDF` jsPDF) → **report
  cumulativo** con evidenza FES per alunno. Considerato completo; questa slice vi aggiunge solo il
  marcatore **cartaceo**.
- `forms_submissions` (legacy): `form_id, parent_id, student_id, answers, is_signed, signature_log,
  pdf_path`. Parent via `legame_genitori_alunni` (tabella reale per scope genitore).
- "Sollecito firme docente" = `handleSendReminder` è un **toast finto** (nessun cron) → non si tocca
  (regola: nessun reminder/cron automatico).

## Decisioni
1. **Upload reale** della scansione su `form_attachments/cartaceo/{form_id}/{uuid}-{nome}` (service-role).
2. **Gate `requireDocente`** (educator/admin/coordinator/segreteria); identità dello staff dall'auth.
3. **Flag `origine`** (`online`|`cartaceo`) su `forms_submissions` (migr. 20260749) per distinguere e
   marcare nel merge.
4. **Evidenza strutturata** (non finge una FES digitale): `signature_log` cartaceo `{ method:
   'PROXY_CARTACEO', acquisito_da: <staffId>, ip, user_agent, timestamp, signed_at, compliance:
   'Acquisizione cartacea validata dallo staff' }` + **audit** `logScrittura('modulistica_cartaceo')`.

## Design
### Migrazione 20260749
`ALTER TABLE forms_submissions ADD COLUMN origine text NOT NULL DEFAULT 'online'` (+ CHECK in/`online`,`cartaceo`).

### `POST /api/teacher/modulistica` (riscritto)
- `requireDocente` → 401/403.
- **multipart**: `file`, `form_id`, `student_id`. 400 se mancano file/form_id/student_id; valida tipo
  (PDF/immagini) + dimensione (≤8MB).
- upload service-role → `pdf_path`; parent via `legame_genitori_alunni`.
- insert `forms_submissions`: `is_signed=true`, `origine='cartaceo'`, `pdf_path`, `signature_log`
  cartaceo, `answers={ proxy:true }`. audit `logScrittura`.
- ritorna `{ success:true, path, id }`.

### UI teacher `/teacher/modulistica`
- Modal proxy: tiene il **File** reale; `handleProxyUploadSubmit` invia **FormData** (file+form_id+
  student_id) con header `x-user-id`. (GET semaforo invariato.)

### Merge PDF (marcatore cartaceo)
- `documents-merge` aggiunge `origine` all'output per alunno; `handleExportMergePDF` stampa
  "(CARTACEO)" accanto allo stato firmato quando `origine='cartaceo'`.

## Cosa NON è in scope
- Solleciti/reminder (cron). Proxy sul sistema moderno `form_models` (qui è il flusso legacy classe).
  Concatenazione reale dei PDF (il merge resta report cumulativo).
