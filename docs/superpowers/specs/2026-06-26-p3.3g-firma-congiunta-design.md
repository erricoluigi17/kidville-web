# P3.3g — Firma congiunta (2° firmatario) + reinvio OTP

**Fase:** P3 · **Slice:** P3.3g · **DL:** DL-031 · **Migrazione:** 20260748
**Branch:** feat/master-plan-registro

## Obiettivo
Completare la firma elettronica dei moduli (Sistema A `form_submissions`) con la **firma congiunta**
(entrambi i genitori) e il **reinvio OTP**, riusando l'infra FEA P1 (slot `fea_signatures`, policy
`all-required` DL-007, `ReceiptPayload.slots`).

## Contesto (verificato)
- `/api/forms/send-otp`: POST crea submission `pending_signature` + invia OTP al genitore loggato;
  PATCH verifica → `completed` + `signature_log` + `recordSignerSlot(slot 0, any-one)` + `logFeaEvent`.
  **Un solo firmatario, completamento immediato.**
- FEA P1: `fea_signatures` (slot per-firmatario), `isComplete(slots, policy)`, `getSlots`,
  `recordSignerSlot` (upsert idempotente per `slot_index`), `ReceiptPayload.slots` già previsti.
- `form_models.requires_signature` bool (derivato dalla presenza di un blocco `signature`). **Nessun**
  concetto di firma singola vs congiunta.
- Il path pubblico `/m/[token]` ha la firma OTP **disattivata** (DL-030): la congiunta vale per il
  **wizard autenticato**.

## Decisioni di prodotto
1. **`signature_mode`** su `form_models`: `single` (default) | `joint`. Toggle nel builder quando è
   presente un blocco Firma.
2. **Firma sequenziale**: firma il 1° genitore (slot 0); se `joint`, la submission resta
   `pending_signature` finché il 2° genitore (slot 1, **email diversa**) non firma. Completamento =
   `all-required` su 2 slot.
3. **Reinvio OTP**: rigenera e re-invia il codice per la submission corrente (stesso firmatario) — con
   **cooldown** lato UI; rate-limit invariato lato server.
4. **2° firmatario**: identità **email-only** (slot `signer_user_id` null ammesso) — evidenza FES
   email+IP+hash, coerente con FEA in-house.

## Design

### Migrazione 20260748
```sql
ALTER TABLE form_models ADD COLUMN signature_mode text NOT NULL DEFAULT 'single'
  CHECK (signature_mode IN ('single','joint'));
```

### Helper puro — `src/lib/fea/firma-congiunta.ts`
- `firmatariRichiesti(mode)` → `joint` ? 2 : 1.
- `firmaCompleta(mode, slotFirmati)` → `slotFirmati >= firmatariRichiesti(mode)`.
- `prossimoSlot(slotFirmati)` → `slotFirmati` (indice del prossimo slot da firmare).

### `POST /api/forms/send-otp` — create | resend | next-signer
- **`submissionId` assente** (create): invariato (crea `pending_signature`, consensi, OTP al genitore).
- **`submissionId` presente** (resend / 2° firmatario): carica la submission (`pending_signature` o
  404/409), genera nuovo codice, salva `otp_secret`, invia a `signerEmail` (se assente → email del
  `user_id`). Ritorna `{ submissionId, email, sent, devCode? }`. Riusa rate-limit.

### `PATCH /api/forms/send-otp` — verify slot + completamento per policy
- Verifica codice (invariato: 404/409/400).
- `slotIndex = prossimoSlot(#slot già firmati)` (via `getSlots('forms', submissionId)`).
- `recordSignerSlot(slotIndex, policy)` con `policy = joint ? 'all-required' : 'any-one'`; `signature_log`
  per-firmatario; per slot 0 imposta anche `form_submissions.signature_log`.
- Carica `form_models.signature_mode` (via `submission.model_id`). Se `firmaCompleta(mode, firmati+1)` →
  `status=completed`, `signed_at`, `otp_secret=null`, `logFeaEvent('signed')`, ritorna
  `{ ok, signedAt, completed:true }`. Altrimenti → `otp_secret=null`, resta `pending_signature`, ritorna
  `{ ok, completed:false, needsMoreSigners:true, signedSlots, requiredSigners }`.
- **Retro-compat:** senza mock di `form_models`/`fea_signatures` (test esistenti) → mode `single`,
  0 slot → completa al 1° codice come prima.

### `/api/admin/form-models` — persistere `signature_mode`
- POST/PATCH accettano e salvano `signature_mode` (default `single`).

### UI
- **Builder**: quando lo schema contiene un blocco Firma, mostra il toggle **"Firma congiunta
  (entrambi i genitori)"** → `signature_mode`. Invia `signature_mode` nel salvataggio.
- **OtpSignatureModal**: bottone **"Reinvia codice"** (cooldown ~30s) → POST send-otp `{submissionId}`.
  In `joint`, alla risposta `needsMoreSigners` mostra lo **step 2° firmatario**: input email →
  "Invia codice al 2° genitore" (POST `{submissionId, signerEmail}`) → verifica codice slot 1 → completato.
- **WizardContainer**: passa `signatureMode` al modal; gestisce il proseguimento finché `completed`.
- Il modello `/parent/forms/[id]` passa `signature_mode` come prop.

## Cosa NON è in scope
- Firma OTP su form **pubblici** (resta disattivata). Più di 2 firmatari. Firma parallela (resta sequenziale).
